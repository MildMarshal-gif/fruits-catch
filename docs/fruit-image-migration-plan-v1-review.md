# Fruit Catch 画像移行実装プラン v1 レビュー

## 1. 判定
- Needs another loop

## 2. Findings（重大度順）

### 中-1: 低速回線/読込遅延時の受け入れ基準が数値化されていない
- 指摘:
  - v1の 6章・11章にフォールバック方針はあるが、タイムアウト閾値や「開始可能までの待機上限」が未定義。
- 影響:
  - 実装者ごとに挙動がぶれ、起動待ちや無限ロードの回帰を招く可能性がある。
- 修正要求:
  - `ASSET_PRELOAD_TIMEOUT_MS`、`MAX_RETRY`、`startGame()` の待機方針（待機/非待機）を明記する。

### 中-2: ロールバック手段（機能フラグ）が欠落
- 指摘:
  - 画像化後に本番不具合が出た場合、既存Canvas描画へ即時切戻しする運用設計がない。
- 影響:
  - 重大不具合時に修正デプロイ待ちとなり、復旧時間が伸びる。
- 修正要求:
  - `USE_IMAGE_SPRITES` / `USE_IMAGE_BASKET` / `USE_IMAGE_FX` のフラグ設計を追加する。

### 軽微-1: QA合格条件の定量指標が不足
- 指摘:
  - 11章で「実用範囲」「許容範囲」とあるが、FPS/発火成功率/クラッシュ再現条件が曖昧。
- 影響:
  - 判定者により合否がぶれる。
- 修正要求:
  - FPS下限、演出発火成功率、連続プレイ時間の閾値を追記する。

## 3. 要件チェック結果
